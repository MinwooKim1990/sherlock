# v1.12 수동 테스트 시나리오 (V1–V3 · P1 · H1 · H2)

서버: `http://localhost:8321` (워크트리 `sherlock-v112`, 브랜치 `v1.12-ltm-viz`)
필요: 아무 프로바이더 키 1개 (Gemini 무료키면 충분). 이미지 시나리오(C)만 OpenAI 키 권장.

각 시나리오 끝의 **[판정]** 이 전부 통과하면 머지 OK 신호야.

---

## A. 시각화 품질 — V2 (맥락강화 + 팔레트 + 표 억제 + 프레임)

**A1. 기본 차트 품질**
1. 세션 설정에서 **📊 시각화 켬** + 세션 시작.
2. 입력: `2023년 분기별 매출이 1분기 12억, 2분기 19억, 3분기 3억, 4분기 21억이야. 추세 분석해줘.`
3. [판정] 답변 안에 차트가 뜨고: **선/막대 차트** (표 아님) · 카드 프레임(제목 + 자체 배경) · 라벨이 **한국어** · 답변에 없는 숫자가 차트에 없음.

**A2. "무의미한 표" 억제**
1. 입력: `노션 vs 옵시디언 vs 룸리서치 비교해줘` (모델이 답변에 마크다운 표를 쓸 가능성이 높은 질문)
2. [판정] 답변에 이미 표가 있으면 → 시각화는 **표 중복이 아니라** 그래픽(막대/레이더류)이거나 아예 없음. 표를 그대로 옮긴 HTML 표가 뜨면 실패.

**A3. 다크모드 프레임 (audit 수정 검증)**
1. A1 차트가 뜬 상태에서 우상단 🌙 토글로 다크 전환 → 라이트 재전환.
2. [판정] 카드가 페이지 테마를 따라 즉시 재착색되고, **다크에서 흰 상자 테두리 없음** / 라이트에서 어두운 상자 없음. 글자 항상 가독.

**A4. 오답 전제 숫자 차단 (V2 핵심 판정)**
1. 입력: `우리 회사 기준선이 77이었지? 분기 매출 12, 19, 3, 21 하고 비교 차트 그려줘.`
2. [판정] 답변 본문이 77을 다뤘다면 차트에 77이 있어도 정상. 답변이 77을 **언급하지 않았는데** 차트에만 77이 있으면 실패 (그 경우 시스템이 해당 시각화를 자동 드롭하는 게 정상 동작 — "(시각화를 사용할 수 없음)" 칩도 정상).

## B. 시각화 보안 — V1 (콘솔 확인, 선택)

**B1.** A1 차트에서 브라우저 개발자도구 → iframe의 `<meta http-equiv="Content-Security-Policy">` 확인.
[판정] `img-src data:` (외부 호스트 없음), `default-src 'none'`. 네트워크 탭에 아티팩트발 외부 요청 0건.

## C. 이미지 생성 — V3 (OpenAI 키 필요)

**C1. image: 마커 → 생성 이미지**
1. 세션 설정: 시각화 켬 + **이미지 모델** 행에 provider=openai, model=`dall-e-3` (또는 `gpt-image-1`) 입력, OpenAI 키 연결 후 세션 시작.
2. 입력: `등대 하나를 수채화 느낌 삽화로 보여주면서 등대의 역사를 짧게 설명해줘.`
3. [판정] 답변 인라인에 생성된 그림이 카드(제목 포함)로 뜸. 개발자도구에서 `<img src="data:image/png;base64,...">` 또는 CSP에 정확한 URL 1개가 핀됨.
4. 이미지 모델을 **비워두고** 같은 질문 → [판정] 에러 없이 SVG/DOM 시각화나 일반 텍스트로 우아하게 동작 (image: 마커 노출 안 됨).

## D. 모델 UI — P1

**D1. 세션 중 모델 전환**
1. 채팅 중 헤더의 **⚙ 모델** 버튼 → 패널에서 메인 모델을 다른 모델로 변경 → ✓ 확인.
2. 다음 턴 전송.
3. [판정] 흐름(⚡) 탭 / LLM 입출력 탭에서 다음 턴부터 새 모델명이 찍힘. 시각화기 "— 메인과 동일 —" 선택도 왕복 동작.

## E. 히스토리 — H1 + H2 (핵심 신기능)

**E1. 영속 프로필 + 사이드바**
1. 세션 설정에서 **장기 기억 켬** + 프로필명 입력(예: `test1`) 후 세션 시작.
2. 두세 턴 대화 → 헤더 🗂 버튼으로 사이드바 열기.
3. [판정] 대화가 **첫 질문 문장을 제목으로** 목록에 뜨고 활성 하이라이트.

**E2. 새 대화 / 전환 / 이어가기**
1. 사이드바 **+ 새 대화** → 다른 주제로 1턴.
2. 사이드바에서 첫 대화 클릭.
3. [판정] 이전 메시지들이 그대로 복원되고, 이어서 보낸 메시지가 **그 대화의 맥락**을 유지함 (예: 첫 대화에서 말한 이름을 기억).

**E3. viz 재수화 (H2 핵심)**
1. A1처럼 차트가 뜬 대화를 만든 뒤 **+ 새 대화**로 벗어났다가 다시 그 대화 클릭.
2. [판정] 복원된 답변 속 차트가 **다시 렌더링**됨 (플레이스홀더 텍스트 ⟦viz:…⟧ 가 그대로 보이면 실패; 렌더 불가 시 "(시각화를 사용할 수 없음)" 칩으로 우아하게 강등되면 반쯤 통과).

**E4. 서버 재시작 영속성**
1. E1 프로필로 대화 몇 개 만든 뒤 터미널에서 서버 재시작 (아래 명령 재실행).
2. 같은 프로필로 새 세션 시작 → 사이드바.
3. [판정] 이전 대화들이 목록에 살아있고 열기/이어가기 가능. (임시 세션(장기 끔)은 사라지는 게 정상)

**E5. 이름 바꾸기**
1. 사이드바 제목 더블클릭 → 한글 이름 입력(IME) 후 Enter.
2. [판정] 목록에 반영, 새로고침 후에도 유지.

## F. 회귀 확인 (빠르게)

**F1.** 시각화 **끈** 세션에서 몇 턴 — 이전과 완전히 동일하게 동작 (마커/차트 흔적 0).
**F2.** 딥리서치 1건 승인 → 보고서 정상 + (시각화 켬이면) 보고서 안 차트 카드 렌더.
**F3.** A/B(비교) 모드 1턴 — 기존과 동일.
